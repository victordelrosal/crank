Stop being the loop. Design the loop.

In June 2026 the AI coding world spent a week arguing about one word. This is the plain version, and a working skill that does it on a real budget: interactive mode, and an unattended Scheduled Mode that runs on a heartbeat.

Strip the mystique and any agent loop is four parts:

  • Generator produces the next prompt (a template, a task list, or a mutation of the last attempt).
  • Agent run is the model call that does the work.
  • Evaluator is a binary or scored check on the output. No eval is not a loop, it is a fancy for.
  • Controller reads the eval and decides: stop, retry, escalate, fan out, or mutate.

The prompt is the cheap part now. The art is almost entirely in the evaluator and the controller. A loop with a weak evaluator fails confidently a thousand times and bills you for it.

One skill, /diy-loop, with two ways to run it. The difference is whether you are in the room.

Interactive mode

/diy-loop (default)

You hand over the wheel and stay present between rounds. A gate pauses the loop in-session and waits for you.

  • Cycle: orient, frame, decompose, execute, red-team, decide
  • Writes binary acceptance criteria before spending tokens
  • A cold, separate agent tries to fail the work each round
  • Stops and asks before anything irreversible

Scheduled Mode

/diy-loop scheduled ...

The same loop with the human pulled out of the room. Launched by cron, it runs while you are away.

  • Gates become an async approval queue, never auto-approved
  • A hard token or dollar ceiling that aborts, not headroom
  • Crash-safe STATE.json so a cold start resumes
  • Does all the safe work, parks the unsafe decisions for you

What stays identical: binary criteria as the contract, adversarial verification before the result reaches you, honest downgrades when a criterion is relaxed, and a hard stop before anything irreversible or outward-facing.

Verification is the moat, not the loop. Everyone is selling the loop. The loop is plumbing. The defensible part is a separate adversary, with context isolation, that is trying to fail the work before it reaches you. Self-verification is the author being too nice grading its own homework.

A good loop should not deskill you. The faster it ships code you did not write, the wider the gap between what exists and what you understand. Every run ends with a short, honest note headed What you would need to do this yourself, so the loop never quietly leaves you more served and less able.

This is not year zero. The honest ladder, oldest to newest:

  • ReAct (2022) formalized reason, act, observe, repeat.
  • AutoGPT (2023) gave it a goal and let it prompt itself, and became famous for spinning forever.
  • The ralph loop (Geoffrey Huntley, 2025) piped the same prompt file in repeatedly, resetting context each iteration; built a working programming language on it for a few hundred dollars of compute.
  • /goal and /loop (2026) productized the ralph loop until a validator confirms done.
  • Orchestration loops (now) where loops supervise other loops, on a schedule, with durable state.

Crank is the closed-loop, budget-disciplined member of that family: a human designs the path, every step has an eval, and there is a defined point where it stops or hands back.

The most common dismissal, and it is half right. The scheduling layer is cron. What cron never had is the part in the middle. A cron job runs a fixed script, the same path every time regardless of the state of the world. A loop runs a model that reads the current state, picks the next action, checks whether it worked, and decides whether to keep going. The decision is the model's, not a hardcoded branch.

So the honest framing is neither "loops are new magic" nor "loops are just cron". It is that loops are cron plus a decision-maker in the body, and the interesting engineering is everything you wrap around that decision so it does not run off a cliff.
